## 07. 추천 (Recommendation)

### 📌 추천 개요
본 서비스의 추천 기능은 **아이템(Item)에 집중한 콘텐츠 기반 추천 방식**을 사용합니다.  
사용자 간 유사도를 계산하는 방식이 아닌, **아이템 간 내용 유사도**를 기반으로 추천을 제공합니다.

해당 방식은 사용자 데이터가 충분하지 않은 초기 서비스 단계에서도 안정적으로 동작할 수 있습니다.

---

### 🎯 추천 방식의 특징
- 사용자 행동 로그가 부족해도 추천 가능
- 아이템 자체의 정보(텍스트)를 기반으로 유사도 계산
- 콜드 스타트(Cold Start) 문제에 비교적 강함
- 추천 결과의 기준이 명확하여 해석이 용이함

---

### 🧠 텍스트 기반 추천 – TF-IDF

아이템의 텍스트 데이터를 수치화하기 위해 **TF-IDF(Term Frequency–Inverse Document Frequency)** 기법을 사용합니다.

#### TF (Term Frequency)
- 특정 문서(아이템) 내에서 **단어가 얼마나 자주 등장하는지**를 나타내는 값
- 문서 내부 중요도를 반영

#### IDF (Inverse Document Frequency)
- 전체 문서 집합에서 **해당 단어가 얼마나 드물게 등장하는지**를 나타내는 값
- 흔한 단어의 중요도를 낮추는 역할

> TF-IDF는  
> - 자주 등장하지만 모든 문서에 흔한 단어의 가중치는 낮추고  
> - 특정 문서에서만 등장하는 단어의 가중치는 높이는 방식입니다.

---

### 📐 추천 처리 흐름
1. 아이템별 텍스트 데이터 수집
2. 텍스트 전처리 (토큰화, 불용어 제거 등)
3. TF-IDF 벡터 생성
4. 아이템 간 유사도 계산
5. 유사도가 높은 아이템을 추천 결과로 반환

---

### 🔍 유사도 계산 방식
아이템 간 유사도는 **Cosine Similarity**를 사용하여 계산합니다.

- 벡터 간 각도를 기준으로 유사도 측정
- 값이 1에 가까울수록 두 아이템의 내용이 유사함
- 유사도 상위 N개의 아이템을 추천 대상으로 선정

---

### 🧪 추천 결과
TF-IDF 벡터와 코사인 유사도를 기반으로,  
선택한 아이템과 **내용적으로 유사한 아이템 목록**을 추천 결과로 제공합니다.

---

### ⚠️ 한계점
- 텍스트 품질에 따라 추천 성능이 크게 좌우됨
- 사용자 개인 취향을 직접적으로 반영하기 어려움
- 단어 빈도 기반이므로 의미적 맥락 이해에는 한계가 있음

---

### 🔮 향후 개선 방향
- 문장 임베딩 기반 추천 모델 도입
- 사용자 행동 데이터와 결합한 하이브리드 추천
- 추천 결과에 대한 설명(Explainability) 강화


---

## 08. 예측 (Prediction)

### 📌 예측 기능 개요
본 프로젝트의 예측 기능은 입력 데이터를 기반으로 **특정 결과 값을 추정하는 지도 학습 기반 예측 과정**을 다룹니다.  
데이터 전처리부터 모델 학습, 예측 결과 도출까지의 **일반적인 머신러닝 예측 파이프라인**을 따릅니다.

---

### 🧩 예측 문제 정의
- 입력 데이터(X)를 기반으로 목표 변수(y)를 예측
- 예측 대상은 수치 또는 범주형 값일 수 있음
- 학습 데이터와 동일한 분포를 가정한 테스트 데이터에 대해 예측 수행

---

### 📊 데이터 처리 흐름
1. 원본 데이터 수집
2. 결측치 처리 및 이상치 정제
3. 피처 선택 및 변환
4. 학습용 데이터와 테스트 데이터 분리

이 과정을 통해 모델 학습에 적합한 형태의 입력 데이터를 구성합니다.

---

### 🧠 모델 학습 과정
- 학습 데이터(train set)를 이용해 모델 파라미터를 학습
- 손실 함수(Loss Function)를 최소화하는 방향으로 최적화
- 검증 데이터(validation set)를 통해 과적합 여부 점검

모델의 구조나 알고리즘 선택보다는, **데이터 전처리와 학습 흐름의 안정성**에 중점을 둡니다.

---

### 🔮 예측 수행
학습이 완료된 모델을 사용하여 새로운 입력 데이터에 대해 예측을 수행합니다.

- 학습에 사용되지 않은 데이터에 대해서만 예측 수행
- 입력 데이터는 학습 시 사용한 전처리 과정을 동일하게 적용
- 예측 결과는 모델 출력값 그대로 반환

---

### 📈 예측 결과 해석
- 예측값은 참고 지표로 활용
- 단일 결과보다는 전체 경향성 파악에 중점
- 예측 결과는 데이터 품질과 학습 데이터 분포에 영향을 받음

---

### ⚠️ 한계점
- 학습 데이터에 존재하지 않는 패턴에 대해서는 예측 정확도 저하 가능
- 데이터 편향이 존재할 경우 예측 결과 역시 편향될 수 있음
- 모델 자체보다 데이터 전처리 품질의 영향이 큼

---

### 🔮 향후 개선 방향
- 추가 데이터 수집을 통한 학습 데이터 확장
- 피처 엔지니어링 고도화
- 모델 성능 평가 지표 다양화
- 예측 결과에 대한 신뢰도 또는 불확실성 정보 제공


---

## 09. LLM (Large Language Model)

### 📌 LLM 활용 개요
본 프로젝트에서는 대규모 언어 모델(LLM)을 활용하여  
**자연어 입력을 처리하고 텍스트 기반 결과를 생성**하는 기능을 제공합니다.

LLM은 사전에 학습된 언어 지식을 바탕으로 입력 문맥을 이해하고,  
확률적으로 가장 적절한 텍스트 응답을 생성하는 역할을 수행합니다.

---

### 🧠 LLM의 역할
LLM은 다음과 같은 보조적 역할로 활용됩니다.

- 자연어 질문에 대한 응답 생성
- 텍스트 요약 및 정리
- 입력 문장의 의미 파악 및 변환
- 기존 시스템 결과를 사용자 친화적인 문장으로 변환

LLM은 **결정 로직의 주체가 아닌, 결과를 보조·설명하는 수단**으로 사용됩니다.

---

### 🔄 처리 흐름
1. 사용자의 자연어 입력 수신
2. 입력 텍스트 전처리
3. LLM에 프롬프트 형태로 전달
4. LLM 응답 수신
5. 응답 결과 후처리 및 출력

이 과정에서 LLM은 외부 데이터베이스를 직접 수정하거나  
시스템 상태를 변경하지 않습니다.

---

### ✍️ 프롬프트 설계 원칙
- 입력은 명확하고 간결한 문장으로 구성
- 불필요한 정보는 최소화
- 출력 형식이 필요한 경우 텍스트로 명시

프롬프트는 **응답 품질에 직접적인 영향을 미치므로**,  
일관된 형식과 목적 중심으로 구성합니다.

---

### 📄 출력 결과의 성격
- 결과는 **확률적 생성 결과**이며 항상 동일하지 않을 수 있음
- 사실 정보가 필요한 경우 별도의 검증 로직이 필요
- 참고용 또는 사용자 인터페이스 보조 용도로 활용

LLM의 응답은 **절대적 정답이 아닌 보조 정보**로 취급합니다.

---

### ⚠️ 한계점
- 최신 정보 또는 내부 데이터에 대한 직접 접근 불가
- 잘못된 정보를 그럴듯하게 생성할 가능성 존재
- 출력 결과의 일관성 보장 불가
- 입력 프롬프트 품질에 의존적

---

### 🔮 활용 시 주의 사항
- 중요한 판단 로직에 단독 사용하지 않음
- 예측·추천 결과의 근거 설명 용도로 제한
- 필요 시 후처리 또는 검증 단계 추가

---

### 🔮 향후 확장 가능성
- 도메인 특화 프롬프트 고도화
- 기존 추천·예측 결과와 결합한 설명 생성
- 사용자 입력 기반 대화형 인터페이스 확장


---

## 10. 생성형 AI (Generative AI)

### 📌 생성형 AI 활용 개요
본 프로젝트에서는 생성형 AI를 활용하여  
**입력 조건에 따라 새로운 콘텐츠를 생성하는 기능**을 제공합니다.

생성형 AI는 기존 데이터를 그대로 조회하거나 복사하는 방식이 아니라,  
학습된 패턴을 바탕으로 **새로운 결과물을 생성**하는 역할을 수행합니다.

---

### 🧠 생성형 AI의 역할
생성형 AI는 다음과 같은 보조 기능으로 활용됩니다.

- 입력 텍스트를 기반으로 한 콘텐츠 생성
- 사용자 요청에 따른 결과물 자동 생성
- 반복적인 콘텐츠 제작 작업의 보조
- 사용자의 아이디어를 구조화된 결과로 변환

생성형 AI는 **결정권을 갖지 않으며**,  
사용자의 입력과 시스템 흐름을 보조하는 역할에 한정됩니다.

---

### 🔄 기본 처리 흐름
1. 사용자 입력 또는 조건 수신
2. 입력 데이터 정제 및 구조화
3. 생성형 AI에 요청 전달
4. 생성 결과 수신
5. 결과 후처리 및 사용자에게 제공

이 과정에서 생성형 AI는  
서비스의 핵심 데이터나 상태를 직접 변경하지 않습니다.

---

### ✍️ 입력 설계 원칙
- 생성 목적이 명확하도록 입력 조건을 구성
- 불필요한 정보는 제외
- 결과 형식이 필요한 경우 사전에 명시

입력 조건은 생성 결과의 품질과 일관성에 큰 영향을 미칩니다.

---

### 📄 생성 결과의 성격
- 결과는 **확률적 생성 결과**로, 매번 동일하지 않을 수 있음
- 생성된 콘텐츠는 참고 자료 또는 초안 용도로 활용
- 최종 결과물로 사용하기 전 검토 과정 필요

생성형 AI의 결과는 **자동 완성물이 아닌 보조 산출물**로 취급합니다.

---

### ⚠️ 한계점
- 생성 결과의 정확성과 완전성 보장 불가
- 입력 조건이 불명확할 경우 결과 품질 저하
- 도메인 특화 정보에는 한계 존재
- 생성 결과에 오류 또는 부정확한 내용이 포함될 수 있음

---

### 🔒 활용 시 주의 사항
- 중요한 의사결정에 단독 사용하지 않음
- 생성 결과에 대한 검증 및 수정 단계 필요
- 사용자에게 생성 결과의 특성을 명확히 고지

---

### 🔮 향후 활용 가능성
- 사용자 입력 패턴을 반영한 생성 방식 고도화
- 기존 추천·예측 결과를 기반으로 한 설명 생성
- 반복 작업 자동화를 위한 생성 기능 확장
